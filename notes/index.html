2026.07.23LOG 008

数据库项目与博客智能体优化

ASSIGNMENT / DATABASE SYSTEM & BLOG AGENT

梳理数据库项目的整体结构，了解后端 Spring Boot、前端 Vue、MySQL 数据库与 Redis 缓存之间的关系，进一步熟悉前后端分离项目的基本运行流程。

完成项目运行环境的检查与配置，使用 JDK 17 运行 Java 后端，使用 Node.js 启动 Vue 前端，并确认 MySQL 和 Redis 服务能够正常连接。

测试系统登录功能，发现前端默认填写的 tony 账号在数据库中不存在；随后重新配置管理员账号，并验证登录接口。

通过本次实践进一步掌握前后端分离项目的部署方法，也认识到端口状态、环境变量、数据库账号和服务启动顺序对项目正常运行的重要性。

针对博客智能体无法正确回答问题的情况接入网页检索功能，使其能够自主查询并整理搜索结果。现阶段工具调用能力仍受限制，但已形成一个轻量级 Agent 工作流。

2026.07.22LOG 007

数据库项目测试与翻译系统优化

ASSIGNMENT / DATABASE SYSTEM & TRANSLATION OPTIMIZATION

针对若依管理系统解决账户无法登录的问题，排查出系统缺少账号注册方式，并积极进行优化。

对藏语、维吾尔语、蒙古语、哈萨克语和朝鲜语等语言的测试文本进行翻译验证，重点分析部分句子翻译生硬、词义错误和语言识别不准确的问题。

完成项目编译和接口测试，确认修改后的翻译结果能够正常输出。

针对藏语被错误识别为维吾尔语的问题进行优化，大幅减少不同语言之间的翻译混乱。

增加混合翻译功能，可直接读取 TXT 文件并识别、翻译，提高处理效率。

将 NLLB 翻译模型由 600M 升级为 1.3B；虽然处理速度有所降低，但提高了翻译准确度。

加入重复惩罚和三元短语去重机制，防止翻译结果重复或乱码，减少后续整理工作量。

2026.07.16LOG 003

翻译体验优化与企业数据核验

ASSIGNMENT / TRANSLATION SYSTEM & ENTERPRISE RESEARCH

针对长文本生成速度过慢的问题，优化 Web 服务的解码速度，将目标输出上限设为 128 个 token。

改善语言翻译系统的启动体验，在长时间没有提示时显示“正在加载解码器”等状态文字。

在项目配置中将 VS Code 的运行时锁定为 JDK 8，避免出现前一天能够运行、第二天却无法启动的问题。

增加一键翻译和翻译历史功能。历史记录只保存在浏览器中，不上传服务器，以保护用户隐私。

在抖音搜索日照地区的企业账号，筛选出 26 个具有明确企业认证主体的账号，逐一核对媒体名称、主页网址、平台账号、粉丝量和认证企业名称。开播时间只录入账号主页明确公布的信息，未公开的内容保持空白。

使用企查查、天眼查和爱企查等网站核对工商注册地址，补充企业所属区县与行政区域，并检查企业名称和抖音认证主体是否一致。

按照直播媒体管理表模板重新录入 26 条日照企业信息，统一省、市、区、完善状态和媒介类型等字段，检查账号网址与企业主体是否重复，并完成格式、数量和准确性核验。

2026.07.15LOG 002

修复翻译调用链与端口冲突

ASSIGNMENT / TRANSLATION SYSTEM & ENTERPRISE RESEARCH

将权威网站查询到的企业信息录入表格，整理抖音、快手平台的企业主体清单，补充已核实企业的统一社会信用代码，修正企业简称和不规范名称，并检查主体信息表与媒体管理表的对应关系。
